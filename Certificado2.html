Proyecto Ventas en C# con SQLServer

Requerimientos Funcionales

La aplicación habilitar a los usuarios finales a: • Comprar, vender y actualizar los stocks de mercadería para los clientes que se encuentren en la base de datos. Además, se puede actualizar la misma mediante una interfaz gráfica que mantiene los datos apropiadamente • Agrega y borra clientes de la base de datos • Modificar los nombres y direcciones de los clientes, pero no así su número de documento • Consulta el precio actual de cualquier stock en la base de datos.


Recursos del proyecto

Está provisto con los siguientes recursos para llevar a cabo el proyecto: • Un servidor en el cual está instalada la base de datos • Múltiples clientes que accederán a la misma con la aplicación con interfaz gráfica. • Un entorno de desarrollo para los programas


Restricciones del proyecto

El primer paso en el proyecto es definir los requerimientos obtenidos en términos de los recursos y la tecnología disponible. Específicamente, debe aplicar restricciones para limitar la funcionalidad del proyecto a la tecnología disponible, recursos y límites de tiempo • Dadas las limitaciones de tiempo, las consultas a la base de datos deberán utilizar únicamente el número de documento, el cual es único o el ID único del stock
